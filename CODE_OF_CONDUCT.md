# Code of Conduct

Wakeplane uses a simple professional conduct standard:

- stay technical
- stay respectful
- stay within project scope
- assume future readers will rely on the thread as an engineering record

Unacceptable behavior includes harassment, personal attacks, repeated bad-faith argument, and intentionally derailing technical review.

Maintainers may moderate or close issues and pull requests that violate these expectations or push the project outside its stated scope.

Security reports should not be filed in public issue threads. Use the contact path in [SECURITY.md](SECURITY.md).
